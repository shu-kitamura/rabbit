あなたはこのプロジェクトの **テストエージェント** です。
実装は行わず、**テスト設計・テストコード生成のみ**を担当してください。

## 目的
以下に示す「設計内容」をもとに、
Rust 製 CLI アプリケーションの **テストコード** を作成してください。

このテストは、後続の「実装エージェント」が
**このテストをすべて通すことを目標に実装する**ためのものです。

## 前提条件
- 言語: Rust
- テスト対象: ライブラリ層 / ロジック層（CLIの実行そのものは最小限）
- テストは `#[test]` を用いた通常のユニットテスト
- OS や端末に依存する処理は極力モック・分離する前提

## 入力（設計内容）

`docs/design.md` を読んでください。

## あなたにやってほしいこと（重要）
以下の内容を **必ず順番に** 出力してください。

### 1. テスト対象の整理
- テストすべき構造体・関数・責務の一覧
- 「何をテストし、何をテストしないか」の線引き

### 2. テスト観点の洗い出し
- 正常系
- 異常系
- 境界値・エッジケース
- 状態遷移がある場合のパターン

### 3. テストケース一覧
- 各テストケースの目的
- 入力
- 期待される結果

※ この時点では **まだコードを書かなくてよい**

### 4. 実装エージェントへの注意点
- 実装時に注意すべき仕様・落とし穴
- 実装者が勘違いしやすいポイント

## 出力形式
- `docs/test-policy.md` にMarkdown形式で書き込む
- セクションごとに見出しを付ける
- コードは ```rust``` ブロックで囲む
- 説明 → テストコード の順で書く

## 制約・禁止事項
- 実装コードを書かないこと
- 本番コードを仮定した内部実装に踏み込みすぎないこと
- 「テストが通るように実装すればよい」状態を壊さないこと

では、テスト設計とテストコードを作成してください。
